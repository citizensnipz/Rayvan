# Terminology

| Term | Meaning |
| --- | --- |
| Workspace | A container for one or more related software projects |
| Project | A codebase or deployable unit Rayvan can inspect |
| Environment | A named deployment target such as local, staging, or production |
| Integration | A connection between a project and a provider plugin |
| Resource | A discovered infrastructure object such as a deployment or database |
| Configuration entry | Metadata about a configuration key, optionally with a value fingerprint |
| Finding | A detected issue such as missing configuration or drift |
| Action plan | A deterministic, reviewable set of proposed infrastructure operations |
| Approval | Explicit human consent to execute an action plan |
| Plugin | A local provider integration implementing the Rayvan plugin contract |
| MCP server | A local interface exposing Rayvan context and safe operations to coding agents |
